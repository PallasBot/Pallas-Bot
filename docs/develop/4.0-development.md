# 4.0 开发约定

> 合流目标：`dev` · [pallas-4.0-slim.md](../architecture/pallas-4.0-slim.md)

## 分支

| 分支 | 用途 |
| --- | --- |
| `feat/4.0-slim` | 本体瘦身、core/extra 矩阵、扩展仓迁出 |
| `feat/persona-reply-style` | 牛格（独立 PR） |
| Pallas-Bot-WebUI `feat/4.0` | 插件来源展示；S6 插件商店 |

## 扩展安装

- **命令行**：`uv sync --extra plugins-duel` 等（P0 已指向独立 Git 仓，见下表）；组合包 `plugins-game`、`deploy-full`
- **WebUI 一键安装**：S6，与 `local/plugins` 并存
- **站点定制目录**：见 [站点定制](../architecture/site-customization-and-updates.md)

## 官方扩展 API（S5）

`GET /pallas/api/.../plugins/official-extensions` 返回按 pip 包聚合的清单（`uv_extra`、`install_cli`、`repository_url`、`bundled_in_repo` 等），供 WebUI 商店（S6）与安装指引使用。

## 配置

```toml
[bootstrap]
# load_bundled_extra_plugins = false   # 4.0 默认，仅 core
# load_bundled_extra_plugins = true    # 迁移期加载 bundled extra
```

或 `PALLAS_LOAD_BUNDLED_EXTRA=1`。

## 本地开发

```bash
uv run nb run                                    # 仅 core
PALLAS_LOAD_BUNDLED_EXTRA=1 uv run nb run         # 含本体 bundled extra（迁移期）
uv sync --extra plugins-duel                      # 从独立仓安装决斗扩展
```

与扩展仓 sibling 联调时，可在 `pyproject.toml` 的 `[tool.uv.sources]` 将对应包改为 `{ path = "../Pallas-Plugin-Duel" }` 后 `uv sync`。

未装 `plugins-protocol` 时：`GET …/instances` 返回 `protocol_extension.installed=false`；WebUI 协议页展示安装指引。

## 分片指标内核化

`pallas_console_metrics` 将迁入 `platform/shard` / `console/metrics`，不再以用户可见插件存在。见 [pallas-4.0-slim.md](../architecture/pallas-4.0-slim.md#分片指标内核化架构路线)。

生成/刷新扩展仓源码：`uv run python tools/bootstrap_extension_repos.py`（从本体 `src/plugins` 同步；README 模板在 `templates/pallas-plugin-extension/readmes/`）。

## 实施顺序

S1 矩阵与加载策略 → **S2** 跨进程解耦（`plugin_coord`）与 P0 扩展仓 → **S3** pyproject extras（P0 已接 Git）→ S4 Docker/CI → S5 官方扩展 API → S6 WebUI 插件商店

## S2：plugin_coord

内核与分片经 `src/features/plugin_coord/` 调用 duel / maa / dream 能力；未安装扩展时安全 no-op，已安装时由插件 `on_startup` 注册实现（lazy import 回退：`pallas_plugin_*` → `src.plugins.*`）。

| 桥接 | 消费者 |
| --- | --- |
| `plugin_coord.duel` | `duel_group` / `duel_qte`、repeater、block、greeting、`arknights_duel_resource` |
| `plugin_coord.maa` | `maa_pending_registry` / `maa_seen_registry` / `maa_hub_routes` |
| `plugin_coord.dream` | `dream_drift` |
| `group_fleet_probe` | repeater fanout、`bot_status/shard_count` |
