# 4.0 开发约定

> 合流目标：`dev` · [pallas-4.0-slim.md](../architecture/pallas-4.0-slim.md)

## 分支

| 分支 | 用途 |
| --- | --- |
| `feat/4.0-slim` | 本体瘦身、core/extra 矩阵、扩展仓迁出 |
| `feat/persona-reply-style` | 牛格（独立 PR） |
| Pallas-Bot-WebUI `feat/4.0` | 插件来源展示；S6 插件商店 |

## 扩展安装

- **local 投放**：见 [站点定制](../architecture/site-customization-and-updates.md)
- **WebUI 一键安装**：S6，与 local 并存
- **命令行**：`uv sync --extra plugins-*`（S3）

## 官方扩展 API（S5）

`GET /pallas/api/.../plugins/official-extensions` 返回按 pip 包聚合的清单（`uv_extra`、`install_cli`、`bundled_in_repo` 等），供 WebUI 商店（S6）与安装指引使用。

## 配置

```toml
[bootstrap]
# load_bundled_extra_plugins = false   # 4.0 默认，仅 core
# load_bundled_extra_plugins = true    # 迁移期加载 bundled extra
```

或 `PALLAS_LOAD_BUNDLED_EXTRA=1`。

## 本地开发

```bash
uv run nb run                                    # 仅 core
PALLAS_LOAD_BUNDLED_EXTRA=1 uv run nb run         # 含 duel/draw 等
```

## 实施顺序

S1 矩阵与加载策略 → S2 首包迁出 → S3 pyproject extras（占位）→ S4 Docker/CI → S5 官方扩展 API → S6 WebUI 插件商店
